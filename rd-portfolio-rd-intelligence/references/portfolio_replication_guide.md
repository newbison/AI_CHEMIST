# Portfolio Replication Guide

## What Can Be Reused Directly

- Workflow structure
- Templates
- Rubrics
- Review gate logic
- Evidence quality scoring
- Fact/extraction/inference/recommendation separation
- Post-experiment learning record format

## What Must Be Customized

- VOC categories
- CTQ targets
- test methods
- competitor lists
- assignee lists
- regulatory claims
- approved materials
- manufacturing constraints
- supplier constraints
- risk files
- regional requirements

## What Must Be Revalidated

- X-Y relationships
- material tradeoffs
- performance targets
- process windows
- aging assumptions
- sterilization effects
- benchmark relevance

## Replication Checklist

1. Clone the skill.
2. Create a portfolio-specific configuration note.
3. Add portfolio competitor list.
4. Add platform-specific CTQs.
5. Add approved test methods.
6. Add known internal failure modes.
7. Add manufacturing constraints.
8. Add supplier qualification rules.
9. Define human reviewers.
10. Run one pilot project and update templates.
